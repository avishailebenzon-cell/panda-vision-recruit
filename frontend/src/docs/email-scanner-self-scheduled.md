# מנגנון סריקת מיילים עצמאי (Self-Scheduled)

## 🔴 הבעיה שהתגלתה

סריקת המיילים **עצרה ב-10 במרץ 2026** ולא פעלה כבר שבוע.

**הסיבה:** המערכת הסתמכה על אוטומציה חיצונית (scheduled automation) שהייתה:
- **מושבתת** או **נכשלה** מבלי שנדע
- **לא אמינה** - תלויה בשירות חיצוני שיכול להיכשל
- **לא שקופה** - קשה לדעת מתי היא רצה בפעם האחרונה

## ✅ הפתרון החדש

יצרנו **מנגנון עצמאי** שמנהל את עצמו ללא תלות באוטומציות:

### 1️⃣ **emailScannerSelfScheduled** - הלב הפועם
```javascript
// פונקציה שמריצה את עצמה מחדש אוטומטית
1. רץ את emailCvScanner (הסורק המקורי)
2. בודק אם יש עוד מיילים
3. אם יש - מתזמן את עצמו לרוץ שוב תוך **1 דקה**
4. אם אין - מתזמן את עצמו לרוץ שוב תוך **5 דקות**
5. חוזר לשלב 1 - **לנצח!**
```

### 2️⃣ **startEmailScanner** - הפעלה
פונקציה שמפעילה את הלולאה האוטומטית:
- מעדכן `self_scheduled_enabled = true`
- מפעיל את `emailScannerSelfScheduled`
- **דורש הרשאת admin**

### 3️⃣ **stopEmailScanner** - עצירה
פונקציה שעוצרת את הלולאה:
- מעדכן `self_scheduled_enabled = false`
- הלולאה תיעצר אחרי הריצה הנוכחית
- **דורש הרשאת admin**

### 4️⃣ **EmailScannerControl** - ממשק משתמש
רכיב UI חדש במסך ניהול:
- מציג סטטוס הסורק (פעיל/מושהה)
- כפתורי הפעלה/עצירה
- מונה ריצות ותצוגת לוגים
- עדכון אוטומטי כל 10 שניות

## 🎯 יתרונות הפתרון

### 1. **עצמאות מלאה**
- אין תלות באוטומציות חיצוניות
- המנגנון מנהל את עצמו בקוד
- לא ניתן "לשכוח" להפעיל אותו

### 2. **אמינות גבוהה**
- אם יש עוד מיילים - רץ מיד (תוך דקה)
- אם אין - ממתין 5 דקות ובודק שוב
- לולאה אינסופית שלא תיעצר לבד

### 3. **שקיפות מלאה**
- רישום כל פעולה ב-`SystemActivityLog`
- ממשק UI שמראה מתי הריצה הבאה
- מונה ריצות להיסטוריה

### 4. **חכם ואדפטיבי**
- זיהוי מיילים ממתין → ריצה מהירה (1 דקה)
- אין מיילים → ריצה רגילה (5 דקות)
- חוסך משאבים כשאין עומס

### 5. **בקרה פשוטה**
- כפתור הפעלה/עצירה במסך ניהול
- סטטוס ריאל-טיים
- לא צריך לגעת בקוד

## 📊 איך לבדוק שזה עובד

### במסך ניהול → קליטת מיילים:
1. **סטטוס**: צריך להיות "✓ פעיל"
2. **ריצה הבאה**: תראה מתי הריצה הבאה (בדקות)
3. **מספר ריצות**: עולה עם כל ריצה
4. **ריצות אחרונות**: לוג של 5 הריצות האחרונות

### בלוג המערכת (activity_log):
- תראה רשומות של "הפעלת סריקת מיילים אוטומטית"
- כל ריצה מתועדת עם תוצאות

## 🚀 שימוש

### הפעלה:
```javascript
// קריאה ראשונית (בוצעה אוטומטית)
await startEmailScanner({})

// עכשיו הסורק רץ לבד כל 5 דקות!
```

### עצירה:
```javascript
// בממשק או בקוד
await stopEmailScanner({})
```

### בדיקת סטטוס:
```javascript
const status = await base44.entities.MailScanStatus.list()
console.log(status[0].self_scheduled_enabled) // true = פעיל
console.log(status[0].self_scheduled_next_run) // מתי הריצה הבאה
console.log(status[0].self_scheduled_run_count) // כמה פעמים רץ
```

## 🛡️ הגנות מפני בעיות

1. **בדיקת דגל enabled** - אם משהו משנה ל-false, הלולאה תיעצר
2. **בדיקת is_running** - אם הסורק כבר רץ, מדלג על הריצה
3. **setTimeout לא-חוסם** - לא מחזיק את הפונקציה פתוחה
4. **רישום מלא** - כל פעולה נרשמת בלוג המערכת
5. **ניהול שגיאות** - אפילו אם יש שגיאה, הלולאה תמשיך

## 🔧 תחזוקה

### שינוי מרווח זמן:
```javascript
await base44.entities.MailScanStatus.update(scanStatus.id, {
  self_scheduled_interval_minutes: 10 // שנה מ-5 ל-10 דקות
})
```

### איפוס מונה ריצות:
```javascript
await base44.entities.MailScanStatus.update(scanStatus.id, {
  self_scheduled_run_count: 0
})
```

## 📝 מה השתנה ב-MailScanStatus Entity

שדות חדשים:
- `self_scheduled_enabled` (boolean) - האם הסריקה העצמאית פעילה
- `self_scheduled_interval_minutes` (number) - מרווח זמן ברירת מחדל (5 דקות)
- `self_scheduled_last_run` (datetime) - מתי רץ לאחרונה
- `self_scheduled_next_run` (datetime) - מתי מתוכננת הריצה הבאה
- `self_scheduled_run_count` (number) - מונה ריצות
